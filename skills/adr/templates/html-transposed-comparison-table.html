Orientation: with few alternatives and many criteria, keep the material template's orientation (alternatives as columns). With many alternatives (⩾ ~6), flip the matrix: alternatives as ROWS, criteria as columns, and the Decision row becomes a rightmost **Decision** column. Either orientation must carry a per-alternative outcome cell. Medium mapping — MkDocs/Markdown rules and their HTML analogs. Reviewers validate against THIS list, not against the Markdown medium: - Frontmatter (R24/R28) → no YAML. Exclusion marking (Stage 2 — if excluded): - Prerequisite rejections that never entered the matrix → a compact "Excluded early" two-column table (Candidate | Why excluded) BEFORE the main matrix. State in one sentence the split rule: early = fails a hard constraint without needing the comparison; matrix = the comparison itself shows the disqualifier. - Rejected finalists stay as matrix rows: add class="excl" on the <tr> (light-red row tint), class="hot" on the disqualifying cell (brighter red), and the Decision cell reads "Excluded — <reason>". Stage 2 — if keep or undecided: plain row; Decision cell carries the outcome ("Keep", "Spike — shortlist #N", "Deferred — <condition>") using the tag spans. Never decide for the user; overall status stays undecided until every Decision cell is closed.
